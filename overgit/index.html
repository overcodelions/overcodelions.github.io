┌─ [ 01 ]  ·  a desktop git client for people who live in many repos

Many repos. One workspace.

Overgit is a desktop git client built around workspaces — named groups of repos you act on together. Branch across api-gateway, billing-svc, and ledger-core in one move. Stage with checkboxes. Pipe diffs to claude, codex, or gemini for review. Every action runs as a plain git command in the repo you already have.

over·git  — a client that sits over your git repos, never inside .git. Stop using it anytime; your repos behave the same in any other tool.

[ 02 ]  ·  inside the window

Built for the day your work spans four repos.

GitHub Desktop, Tower, and Fork are all one-repo-at-a-time. If your work spans services + shared libs + infra, you end up scripting your own "checkout feature/x everywhere" flow. Overgit fills that gap — without ever writing inside .git.

01

Workspaces, not repos

Group related repos into a named workspace. See status, branches, dirty state, and open PRs across every member at once. The polyrepo problem, finally a UI.

02

Workspace-wide branching

Sync each repo to its default branch → pull → branch — all in one workflow. Per-repo outcomes if any of them fail. Stop scripting for d in */; do git checkout ….

03

Overlay, not metadata

Overgit never writes inside .git. No hidden state, no .overgit folder, no special config. Stop using it any time and your repos behave the same in any other tool.

04

Branch picker, with brains

⌘B opens a searchable popover. Local + remote branches grouped, ↑↓/Enter, inline create-branch with optional sync-and-pull, per-branch cherry-pick with commit-list multi-select.

05

Command palette

⌘K opens a context-aware palette: switch / create branches, jump to repos and workspaces, search files, run repo actions (stage all, fetch, pull, push, AI review).

06

AI review & suggest

Detects claude, codex, and gemini. Pipes diffs into the chosen CLI in non-interactive mode. Suggest drafts a conventional-commit message from the staged diff and drops it into the commit input.

07

Per-repo tabs

Changes / History / Files / Graph / Stash. Multi-select with checkboxes, bulk action bar (Stage / Stash / Discard). File editor sandboxed to registered repos with hljs syntax highlighting.

08

gh PR aggregation

Workspace-wide PR list via gh pr list --json per member, merged into one view. See what's open across the whole workspace, not one repo at a time.

09

Branch graph

Per-lane colored visualization with ref labels. Resizable, readable, no Gitk vibes.

10

Your CLI auth, your machine

Whatever auth your claude, codex, gemini, and gh already have is the auth Overgit uses. Nothing leaves your machine via Overgit.

11

Keyboard first

⌘K palette. ⌘B branches. ⌘N new branch. ⌘1–4 tab switch. ⌘R refresh. ⌘\ sidebar. ⌘, settings. It is not a website; it is a tool.

12

Light, dark, system

Theme follows the OS by default, override per app. Resizable sidebar, sensible defaults, no surprise modals.

[ 03 ]  ·  the workspace move

One branch. Many repos. One commit flow.

A workspace branch is just a name overgit applies across every member repo. Each repo gets the same feature/x via a plain git checkout -b. When you push, you push them all. When you open PRs, gh opens them all. The repos never know they're related — overgit knows.

[ 04 ]  ·  why overgit exists

Coordinate the repos. Own none of them.

Most multi-repo tooling solves the problem by inventing a meta-format: a manifest, a sub-tree, a worktree-of-worktrees. Then you can't open one of the repos in another tool without something breaking. Overgit refuses that bargain. Workspaces are a UI grouping, not a file on disk. Every action is a plain git command run in the repo's own directory. If you uninstall overgit tomorrow, your repos forget it ever existed.

The result is a client built for the realistic case — your work spans services and shared libs and infra, your teammates use whatever client they like, and the only thing the repos agree on is git. Overgit sits over the top, coordinates, and stays out of the way.

Sibling project of overcli. Written & maintained by Lionel Farr and Owen Farr. Open-source — contributors, issues, and PRs welcome.
