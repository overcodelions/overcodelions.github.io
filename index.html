01

overcli

four coding agents, one honest window

↗

A desktop GUI over claude, codex, gemini, copilot, and ollama. Agents work in isolated git worktrees you can rebase, merge, or throw away from inside the conversation — no terminal context-switch. Conversations, diffs, permission flows, and usage stats in one window. Uses the auth you already have — your Claude Pro, ChatGPT plan, gcloud creds, or GitHub Copilot seat. No API keys, no new bills, no telemetry.

  • multi-backend chat with streaming diffs
  • workspaces — projects of projects
  • agent worktrees — create, rebase, merge from the chat
  • silent background agents

02

overgit

workspaces for repos, worksets for tickets

↗

A desktop git client built on two orthogonal ideas. Workspaces are durable groupings of repos — fetch all, reset all, status all. Worksets are tickets you can resume across repos, pinned to a feature branch — spin up, ship, archive, reset. Smart safety on every destructive flow. Never writes inside .git; uninstall any time and your repos forget overgit ever existed.

  • workspaces — fetch all · reset all · status all
  • worksets — branch · commit · push · archive · reset
  • smart safety on every destructive flow
  • AI review & commit-message suggest · gh PRs aggregated
v0.1 · beta macOS · win · linux Apache 2.0 open overgit →

[ why ]  ·  the bar we hold ourselves to

Built to be explainable, not clever.

Codelions tools optimize for the fourth afternoon — when you've shipped three things with one, hit a weird edge, and need to understand what happened. Every surface exists to make that moment easier: diffs you can read, tool cards you can read again, permission flows you can follow, history that came from a file on disk.

No telemetry. No hidden state. No invented abstractions. If something is on screen, there's a line of code you can find that put it there. That's the bar.

Built by Lionel Farr and his son Owen Farr under Codelions. Open-source — issues and PRs welcome on both repos.

[ the longer story ]  ·  how codelions actually started

Owen wanted to build for real. Terminals were a wall. So we built around them — for both of us.

Owen is my son. He'd been making things on his own — toy projects, half-finished games, the usual kid-with-a-laptop stuff. Nothing that left the disk. So I started teaching him git, and then how to use Copilot and Claude, because LLM tools are the most leveraged way for a beginner to actually ship something real.

He took to the LLMs fast. He didn't take to the editor. He wanted to talk to the code, not navigate around it. The honest path for that is the coding-agent CLIs — claude, codex, gemini — but a terminal is not a place a kid learns naturally. Output scrolls away. Cursor keys behave oddly. There's no "scroll back, the answer was a screen ago." The tool that should have unlocked everything was the thing in the way.

So we built overcli — a GUI for those CLIs that worked for both of us, not just him. Diffs you can read. Conversations you can scroll. Permission prompts that are buttons. The terminal is still there underneath; the friction isn't. Overgit followed once the work started spanning multiple repos and we hit the same kind of wall again, this time around git.

Two things came out of it, and both are why this is worth doing. Owen got to build real software — bug reports from real users, cross-platform builds that fail in obscure ways, gatekeeper warnings, license headers — texture no tutorial fakes. And the code itself ended up better for it, because every weird abstraction is a question you'd have to answer to Owen. Explainable, not clever isn't a slogan up here; it's the only version of the code that survives the next conversation downstairs.

— Lionel & Owen Farr
