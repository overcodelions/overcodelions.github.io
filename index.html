Two tools. One philosophy.

Codelions builds small, honest desktop apps for developers who already live in CLIs. Apps you can read, audit, and uninstall without leaving a trace.
Right now there are two of them — and they're sibling projects on purpose.

01

overcli

four coding agents, one honest window

↗

A desktop GUI over claude, codex, gemini, and ollama. Conversations, diffs, agent worktrees, permission flows, colosseum mode, usage stats — every coding-CLI surface in one window. Uses your existing CLI auth. No API keys, no new bills, no telemetry.

  • multi-backend chat with streaming diffs
  • workspaces — projects of projects
  • colosseum — fire one prompt at every backend
  • silent background agents
v0.1 · beta macOS · win · linux Apache 2.0 open overcli →
02

overgit

many repos, one workspace

↗

A desktop git client built around workspaces — named groups of repos you act on together. Branch across services in one move. Stage with checkboxes. Pipe diffs to claude, codex, or gemini for review. Never writes inside .git; uninstall any time and your repos forget overgit ever existed.

  • workspace-wide branch / pull / push
  • command palette · branch picker · graph
  • AI review & commit-message suggest
  • gh PR aggregation across repos
v0.1 · beta macOS · win · linux Apache 2.0 open overgit →

[ why ]  ·  the bar we hold ourselves to

Built to be explainable, not clever.

Most developer tooling optimizes for the demo. Codelions tools optimize for the fourth afternoon — when you've shipped three things with one, hit a weird edge, and need to understand what happened. Every surface in our apps exists to make that moment easier: diffs you can read, tool cards you can audit, permission flows you can follow, history that came from a file on disk.

The result is software that trades a bit of polish for a lot of legibility. No invented abstractions. No hidden state. No telemetry. If something is on screen, there's a line of code you can find that put it there. That's the bar.

Built by Lionel Farr and Owen Farr at Codelions. Open-source — issues and PRs welcome on both repos.
